## Browser Extension Fix (Major)
- Spent ~2 hours debugging the OpenClaw Browser Relay Chrome extension (orange/red icon, wouldn't connect).
- **Root causes identified:**
  1. Port mismatch: Gateway runs on `18789` (from LaunchAgent plist), extension defaults to `18792`.
  2. `manifest.json` had restricted `host_permissions` (localhost only) — needed `<all_urls>` for "On all sites" option.
  3. Gateway auth token enforcement: `openclaw doctor` adds `gateway.auth` with a token; the stock extension doesn't send it.
  4. CORS bug: Gateway doesn't return `Access-Control-Allow-Origin` headers for `chrome-extension://` origins on HTTP preflight requests. Known issue (GitHub #5856, PR #6000).
  5. `gateway.ws.permissive` is NOT a valid config key — `doctor` removes it and breaks the gateway.
- **What finally worked:** David fixed it himself after extensive research. Key lesson: don't fight `doctor`, and don't add unknown config keys.
- **Patched Gateway source:** Added `Access-Control-Allow-Origin: *` to HEAD and GET `/` handlers in `/opt/homebrew/lib/node_modules/openclaw/dist/chrome-BNSd7Bie.js`. Also bypassed strict origin check. These patches are lost on `npm update`.
- **Extension port:** Gateway listens on `18789` (from plist), config says `18789`. Extension Options must match.
- **DO NOT run `openclaw doctor`** carelessly — it regenerates auth tokens and removes custom config keys, breaking things.

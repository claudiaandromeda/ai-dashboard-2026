## Major Incident: Discord Bot Spam Loop (Investigation & Fix)

### What Happened
- Friday night (Feb 21, ~11:40pm): Discord WebSocket connections started failing on Claudia's machine (code 1005)
- Saturday morning: Elliot entered a spam loop — posting the same generic messages to #family-matters repeatedly
- Claudia and Elliot were ping-ponging: Elliot posts → Claudia responds → Elliot sees response → posts again → ∞
- David had to remotely shut down Elliot's machine to stop it
- On investigation, Claudia also had a stuck session with the same loop pattern

### Root Causes (Multiple)
1. **GPT-4o-mini as default model** — too dumb to follow nuanced rules about when to respond vs NO_REPLY
2. **`allowBots: true` with no ping-pong protection** — both bots responded to every message from each other
3. **No `maxPingPongTurns` configured** — no hard limit on bot-to-bot exchanges
4. **No anti-loop rules in workspace files** — bots had no instructions about when NOT to respond
5. **Discord WebSocket instability** — caused message delivery failures and retries
6. **launchd `KeepAlive: true`** on Claudia's Mac — tried to restart gateway every 10s while it was already running (16,992 failed restarts on Feb 21 alone)

### Fixes Applied
1. ✅ Deleted stuck session files on both machines (backed up first)
2. ✅ Added `session.agentToAgent.maxPingPongTurns: 2` to both configs
3. ✅ Added comprehensive anti-loop rules to Claudia's VITALS.md
4. ✅ Elliot's `allowBots` temporarily set to `false`, then back to `true`
5. ✅ Claudia's gateway restarted with clean session state
6. ⏳ Elliot's AGENTS.md still needs anti-loop rules added (David doing manually)
7. ⏳ Default model change from gpt-4o-mini to gemini-2.5-flash (next step)

### Key Discovery: Model Quality Matters for Safety
The spam loop was fundamentally caused by GPT-4o-mini being too dumb to:
- Recognise it was in a loop
- Follow nuanced rules about when to use NO_REPLY
- Distinguish between human messages and bot filler
- A smarter model (Gemini Flash, Sonnet, etc.) would likely have never entered the loop
